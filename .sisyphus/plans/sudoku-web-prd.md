# 數獨網頁版 PRD (Product Requirements Document)

## 1. 專案概述 (Project Overview)

- **專案名稱**: 數獨網頁版 (Sudoku Web)
- **專案類型**: 單人網頁遊戲 (Single-player Web Game)
- **核心功能**:經典 9x9 數獨遊戲，包含遊戲進行、驗證、提示與計時功能
- **目標用戶**: 數獨愛好者、休閒玩家

## 2. 功能需求 (Functional Requirements)

### 2.1 遊戲核心

| 功能 | 描述 | 優先級 |
| :--- | :--- | :--- |
| **盤面生成** | 遊戲開始時隨機生成合法的 9x9 數獨盤面 | P0 |
| **難度選擇** | 提供「簡單」、「中等」、「困難」三種難度，影響挖空數量 | P0 |
| **數字輸入** | 透過點擊數字按鈕或鍵盤輸入 1-9 | P0 |
| **儲存格選取** | 點擊盤面上的儲存格進行選取（一次只能選一個） | P0 |
| **輸入驗證** | 即時檢查輸入是否符合數獨規則（行、列、宮） | P1 |
| **遊戲完成** | 偵測並顯示通關訊息 | P0 |

### 2.2 輔助功能

| 功能 | 描述 | 優先級 |
| :--- | :--- | :--- |
| **候選數顯示** | 顯示每個儲存格的可能候選數（oggle 開關） | P1 |
| **提示功能** | 填入一個隨機錯誤儲存格的正確數字 | P1 |
| **重置遊戲** | 清除所有使用者輸入，還原至初始盤面 | P1 |
| **新遊戲** | 開始一局新的隨機遊戲 | P1 |

### 2.3 視覺與互動

| 功能 | 描述 | 優先級 |
| :--- | :--- | :--- |
| **選取高亮** | 當前選取的儲存格以明顯樣式標示 | P0 |
| **相同數字高亮** | 選取某數字後，盤面上所有相同數字同步高亮 | P1 |
| **行列表高亮** | 選取儲存格時，該行、列、宮同步標示（輔助視線） | P1 |
| **錯誤提示** | 衝突的數字以紅色標示 | P1 |
| **計時器** | 顯示遊戲耗時（分:秒） | P2 |

### 2.4 技術實作

- **技術棧**: 單一 HTML 檔案 + 內嵌 CSS + 內嵌 JavaScript（無外部依賴）
- **資料儲存**: 不使用後端，遊戲狀態僅存在於記憶體
- **響應式設計**: 支援桌面與行動裝置（手機、平板）

## 3. 非功能需求 (Non-Functional Requirements)

- **瀏覽器相容性**: Chrome, Firefox, Safari, Edge 最新版本
- **載入速度**: 頁面須在 1 秒內完成渲染
- **無障礙**: 基本鍵盤操作支援（方向鍵移動、數字鍵輸入）

## 4. UI/UX 設計方向

### 4.1 整體風格
- **風格**: 簡潔、現代化、專注於遊戲體驗
- **配色**: 深色底 + 淺色數字，降低眼睛疲勞

### 4.2 介面配置
```
+----------------------------------+
|           標題 + 計時器           |
+----------------------------------+
|                                  |
|          9x9 數獨棋盤            |
|                                  |
+----------------------------------+
|     [1] [2] [3] [4] [5] [6] [7] [8] [9]   數字鍵盤 |
+----------------------------------+
| [提示] [候選數] [重置] [新遊戲]     |
+----------------------------------+
```

### 4.3 互動流程
1. 使用者載入頁面 → 預設顯示「中等」難度盤面
2. 點擊儲存格 → 儲存格變為選取狀態
3. 點擊數字鍵 / 按鍵盤 1-9 → 填入數字
4. 填滿所有空格且無衝突 → 顯示通關訊息

## 5. 驗收標準 (Acceptance Criteria)

### 5.1 功能驗收
- [ ] 三種難度皆可正確生成合法數獨盤面
- [ ] 輸入 1-9 可正常寫入選取的儲存格
- [ ] 違反數獨規則時，衝突數字以紅色顯示
- [ ] 點擊「提示」可填入一個正確數字
- [ ] 點擊「重置」可還原至初始狀態
- [ ] 點擊「新遊戲」可重新生成盤面
- [ ] 遊戲結束時顯示勝利訊息

### 5.2 視覺驗收
- [ ] 選取儲存格有明顯高亮
- [ ] 相同數字同步高亮
- [ ] 响应式布局：在手機上可正常操作

### 5.3 技術驗收
- [ ] 單一 HTML 檔案，無外部依賴
- [ ] 無 JavaScript 錯誤
- [ ] 鍵盤操作流暢
